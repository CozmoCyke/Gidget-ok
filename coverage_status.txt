3. Top priorités à améliorer ensuite

ensure - mot de contrôle important, mais encore trop abstrait pour un débutant
set - très fréquent pour l’état et les variables, il gagnerait à parler d’exemples robots concrets
property - central pour analyze et les tests, mais encore trop dictionnaire
position - utile pour la navigation, manque d’exemple de déplacement réel
return - clé pour les fonctions, souvent source de confusion
name - très fréquent dans les erreurs de ciblage et de boucle
create - utile pour les définitions, mais encore trop générique
energy - visible en jeu, donc une aide plus orientée “échec de niveau” serait utile
right - trop vague pour un mot qui peut parler de direction ou d’alignement
done - bon candidat pour une formulation plus “objectif atteint / état final”
